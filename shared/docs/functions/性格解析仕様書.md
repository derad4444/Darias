# 性格解析システム 仕様書

**ステータス**: 全フェーズ実装済み（週次スケジューラーはデプロイ要）
**作成日**: 2026-05-09
**最終更新**: 2026-05-16（キャラクター成長システム追加 / フェーズ別チャット返答システム追加: signalCount連動で3段階返答深度切り替え / 性格リセット時にsessionChatCountProviderも0リセット）

> 旧称「性格診断リニューアル設計書」。BIG5 100問方式を廃止し、チャットから継続的に性格を自動推定する新システムの完全仕様。

## 概要

チャットを続けるだけで自動的に性格タイプが解析されるシステム。「100問答える」作業は不要。アプリを使うほど自然に性格が浮かび上がる体験。

- 性格解析に必要なチャット数: **30回以上**（シグナル30件到達で元素・タイプ名が確定）
- 性格タイプ: 9元素 × 各2役割 = 18タイプ
- 旧BIG5診断（`confirmedBig5Scores`）は既存ユーザーの後方互換として残存するが、新規ユーザーへの提供は停止

---

## 5つの性格軸

すべてのタグが5軸すべての計算に使われる。元素・役割の判定はその結果から行う。

| 軸 | フィールド名 | -1.0（左端） | +1.0（右端） |
|----|------------|------------|------------|
| エネルギー | `energy` | 内向的 | 外向的 |
| 判断基準 | `judgment` | 感情重視 | 論理重視 |
| 関わり方 | `relationship` | 独立 | 協調 |
| 行動スタイル | `lifestyle` | 自由奔放 | 計画的 |
| 処理スタイル | `processing` | 直感 | 分析 |

---

## 軸スコアの計算方法

```
axis_score =
  (そのタグの+方向件数の重み合計) - (そのタグの-方向件数の重み合計)
  ─────────────────────────────────────────────────────────────
  (その軸に関連する全タグの重み合計)

※ 直近200件 or 直近90日のシグナルを対象
※ 時間減衰あり（decay = 0.95^経過日数）
→ 結果は必ず -1.0〜+1.0 に収まる
```

**更新タイミング**: `signalCount % 10 === 0` のとき（10シグナルごと）

## 9元素システム

### 元素の対応関係

外向と内向が鏡になる対称構造。対の元素は相性表の軸となる。

```
外向グループ               内向グループ
  炎（感情）       ↔         水（感情）
  風（判断中間）   ↔         土（判断中間）
  雷（論理×直感）  ↔         氷（論理×直感）
  光（論理×分析）  ↔         闇（論理×分析）

           無（エネルギー中間・真の中心）
```

### 各元素のイメージ

| 元素 | 性格イメージ |
|------|------------|
| 炎 | 情熱・衝動・感情が外に爆発する |
| 風 | 自由・好奇心・軽やか・感情にも論理にも縛られない |
| 雷 | 決断力・瞬発力・直感的な論理で外へ放電 |
| 光 | 明晰・計画的・分析的な論理で外を照らす |
| 水 | 共感・深さ・感情が内に深まる |
| 土 | どっしり安定・内に根を張る・ぶれない |
| 氷 | 冷静・冷たい本能・直感的な判断を内で凍てつかせる |
| 闇 | 内省・思索・分析的な論理を内に深める |
| 無 | 形がない・全てを受け入れる・観察者 |

### 元素カラー定義

| 元素 | カラー（HEX） |
|------|-------------|
| 炎 | `#E53935`（赤） |
| 水 | `#1E88E5`（青） |
| 風 | `#43A047`（ティール） |
| 土 | `#8D6E63`（ブラウン） |
| 氷 | `#4FC3F7`（ライトブルー） |
| 雷 | `#FDD835`（黄） |
| 光 | `#FFB300`（ゴールド） |
| 闇 | `#6A1B9A`（ダークパープル） |
| 無 | `#9E9E9E`（グレー） |

### 対の元素と相性

| 対 | 共通点 | 対立点 | 相性の性質 |
|----|--------|--------|----------|
| 炎 ↔ 水 | 感情重視 | 外に爆発 vs 内に深まる | 感情で深く繋がるが、ぶつかることも |
| 風 ↔ 土 | 判断が中間 | 自由に動く vs 動かない | 完全な対極、刺激し合うか相容れないか |
| 雷 ↔ 氷 | 論理×直感 | 熱く放電 vs 冷たく凍てつく | 同じ直感型、熱量の違いで惹かれ合う |
| 光 ↔ 闇 | 論理×分析 | 外を照らす vs 内を深める | 同じ分析型、視点の向きが逆 |
| 無 | ― | ― | 全元素と一定の相性（形がなく相手に合わせられる） |

## 元素判定ロジック

### Step 1：元素の決定（3軸使用）

```
エネルギー軸が中間（-0.3〜+0.3）
  → 無

エネルギー軸が外向（>+0.3）
  ├ 判断基準 < -0.3（感情重視）           → 炎
  ├ 判断基準 -0.3〜+0.3（中間）           → 風
  └ 判断基準 > +0.3（論理重視）
      ├ 処理スタイル < 0（直感寄り）       → 雷
      └ 処理スタイル >= 0（分析寄り）      → 光

エネルギー軸が内向（<-0.3）
  ├ 判断基準 < -0.3（感情重視）           → 水
  ├ 判断基準 -0.3〜+0.3（中間）           → 土
  └ 判断基準 > +0.3（論理重視）
      ├ 処理スタイル < 0（直感寄り）       → 氷
      └ 処理スタイル >= 0（分析寄り）      → 闇
```

### Step 2：役割の決定（元素ごとに使う軸が異なる）

| 元素 | 役割判定に使う軸 | 条件 | タイプ名 |
|------|--------------|------|---------|
| 炎 | 関わり方 | < 0（独立） | 独り燃える炎タイプ |
| 炎 | 関わり方 | >= 0（協調） | 場を沸かす炎タイプ |
| 風 | 行動スタイル | < 0（自由） | 自在に舞う風タイプ |
| 風 | 行動スタイル | >= 0（計画） | 目的へ吹く風タイプ |
| 雷 | 関わり方 | < 0（独立） | 単独で閃く雷タイプ |
| 雷 | 関わり方 | >= 0（協調） | 場を揺らす雷タイプ |
| 光 | 行動スタイル | >= 0（計画） | 道を照らす光タイプ |
| 光 | 行動スタイル | < 0（自由） | 先を駆ける光タイプ |
| 水 | 行動スタイル | < 0（自由） | 形を変える水タイプ |
| 水 | 行動スタイル | >= 0（計画） | 深く湛える水タイプ |
| 土 | 関わり方 | >= 0（協調） | 静かに支える土タイプ |
| 土 | 関わり方 | < 0（独立） | 独り立つ土タイプ |
| 氷 | 関わり方 | < 0（独立） | 研ぎ澄ます氷タイプ |
| 氷 | 関わり方 | >= 0（協調） | 凛として守る氷タイプ |
| 闇 | 行動スタイル | < 0（自由） | 深く問い続ける闇タイプ |
| 闇 | 行動スタイル | >= 0（計画） | 先を読む闇タイプ |
| 無 | 関わり方 | >= 0（協調） | 全てに溶け込む無タイプ |
| 無 | 関わり方 | < 0（独立） | 何にも染まらぬ無タイプ |

## キャラクター画像との連携

5軸スコアをBIG5スコアに変換して既存の画像システムを継続使用する。新しい画像生成は不要。

```
Extraversion(E)      = (エネルギー軸 + 1.0) / 2.0 × 4.0 + 1.0
Conscientiousness(C) = (行動スタイル軸 + 1.0) / 2.0 × 4.0 + 1.0
Agreeableness(A)     = (関わり方軸 + 1.0) / 2.0 × 4.0 + 1.0
Openness(O)          = (-処理スタイル軸 + 1.0) / 2.0 × 4.0 + 1.0  ※反転
Neuroticism(N)       = (-判断基準軸 + 1.0) / 2.0 × 4.0 + 1.0      ※反転

結果はすべて 1.0〜5.0 に収まり、既存のL/M/H変換に渡せる
```

変換結果は `details/current.convertedBig5Scores` に保存。

**画像変更のタイミング制限:**
- `signalCount < 30` かつ BIG5 診断未完了（`analysisLevel == 0`）の場合は画像を変更しない
- 理由: 30件未満は軸スコアの変動が大きく、画像が頻繁に変わることでユーザー体験が損なわれる

---

## 変動の安定性

| メッセージ数 | 安定度 | 想定される変動 |
|------------|-------|-------------|
| 〜30件 | 低い | 元素・役割どちらも変動しやすい |
| 30〜100件 | 中程度 | 役割は変動、元素はほぼ安定 |
| 100件以上 | 高い | 元素は安定、役割はごく稀に変動 |

1回の更新（10件）で軸スコアが動く幅は最大0.05程度（200件ウィンドウの5%）。

## generateCharacterReply への5軸反映

`generateCharacterReply` はキャラクター返答生成時に性格特性テキストをプロンプトに渡す。  
`axisScores` が存在する場合は `buildPersonalityTraitsFromAxes()` で生成した5軸ベーステキストを優先使用し、存在しない場合は旧来の `buildPersonalityTraits(big5)` にフォールバックする。

```javascript
// templates.js
function buildPersonalityTraitsFromAxes(axisScores, element, typeName) {
  // 各軸の値からフレーズを選択・連結
  // 例: energy>0.3 → "外向的で人との交流から活力を得る"
  // 例: judgment<-0.3 → "感情や直感を重視して判断する"
  // typeName が存在する場合: "場を沸かす炎タイプ。外向的で..." の形で先頭に付与
}
```

### フェーズ別チャット返答との連携（2026-05-16 追加）

性格データの充実度（`signalCount`）に応じて、返答の深度（フェーズ）を切り替える。

| signalCount | 返答フェーズ | 性格反映 |
|------------|------------|---------|
| < 30（元素未確定） | Phase 1 固定 | **なし**（性格データ不十分のため誤言及を防ぐ） |
| ≥ 30（元素確定済み） | session turn に応じて P1〜P3 | **あり** |

- **Phase 2**（session turn 2〜4）: 性格特性を「仮説」として提示。「〜な面が出てる気がする」など
- **Phase 3**（session turn 5〜）: 性格を言語化して説明。`typeName`（「炎タイプ」等）を自然に使う
- フェーズはFlutter側（`home_screen.dart`）で算出し、`phase` パラメータで Cloud Function に渡す
- `sessionChatCountProvider`（`chat_provider.dart`）でセッション内ターンを管理。アプリ起動時と性格リセット時に 0 にリセット

## 週次ナラティブ生成（Cloud Scheduler）

毎週日曜2:00 JSTに全アクティブユーザーの性格ナラティブを再生成する。

- **対象**: `element` フィールドが存在するユーザー
- **取得**: `collectionGroup('details').where('element', 'in', VALID_ELEMENTS)`
- **生成**: `「最近〜な傾向があります」`形式の100〜150文字テキスト（gpt-4o-mini）
- **保存先**: `details/current.personalityNarrative`・`narrativeUpdatedAt`
- **変化検知**: `element` または `typeName` が前回と異なる場合、`personalityMeta/current.pendingTypeChangeNotification = true` をセット

## キャラクター成長システム

### 概要

`signalCount` に連動してキャラクターが3段階に成長する。成長段階ごとにローカルアセット画像を切り替え、元素確定（signalCount >= 30）後は属性別エフェクトを表示する。

### 成長段階と画像アセット

| ステージ | signalCount | ファイル | 備考 |
|---------|------------|---------|------|
| 赤ちゃん | 0〜29 | `assets/images/character_growth/赤ちゃん.png` | 全ユーザー共通 |
| 幼少期 | 30〜99 | `assets/images/character_growth/幼少期_{元素}.png` | 9元素 × 1枚 |
| 大人 | 100以上 | `assets/images/character_growth/成人_{性別}_{元素}.png` | 2性別 × 9元素 = 18枚 |

アセット合計: 1 + 9 + 18 = **28枚**

### パス生成関数（`characterGrowthAssetPath`）

`lib/presentation/widgets/character/element_effect_widget.dart` に定義。

```dart
String characterGrowthAssetPath({
  required int signalCount,
  required String? element,
  required String? gender,
}) {
  if (signalCount < 30 || element == null) return '赤ちゃん.png';
  if (signalCount < 100) return '幼少期_$element.png';
  final g = gender == '男性' ? '男性' : '女性';
  return '成人_${g}_$element.png';
}
```

### 元素エフェクト（`ElementEffectWidget`）

`signalCount >= 30`（元素確定後）に、キャラクター画像の背後に属性別アニメーションエフェクトを表示する。

- **表示画面**: ホーム画面・キャラクター詳細画面
- **非表示画面**: カレンダー画面（軽量化のため除外）

### キャラクターアバター（`CharacterAvatarWidget`）

設定画面アカウントカード・フレンド一覧・フレンド検索で使用する丸アバター。

- **表示方式**: `ClipOval` + `OverflowBox(maxHeight: size * 1.8, alignment: Alignment.topCenter)` で1.8倍ズームして上部（顔部分）をトリミング表示
- **自分のアバター**: `signalCountProvider` + `characterDetailsProvider` をリアルタイム購読
- **フレンドアバター**: `userCharacterDetailsProvider(userId)` から `growthStage` と `element` を取得し段階を決定
- **フォールバック**: `growthStage == 0` かつ `element != null` → 幼少期として扱う（growthStage 書き込み前の既存ユーザー向け）

## Firestoreスキーマ（新規追加分）

### `users/{userId}/personalitySignals/{signalId}`（新規）

1回のメッセージから抽出したタグを1ドキュメントとして保存する。

```json
{
  "tags": ["social_reference", "emotional_expression"],
  "timestamp": "Timestamp",
  "messageType": "chat"
}
```

### `users/{userId}/personalityMeta/current`（新規）

```json
{
  "signalCount": 42,
  "pendingTypeChangeNotification": false
}
```

- `signalCount`: 累積シグナル数。`savePersonalitySignal` がトランザクションでインクリメント
- Flutter `signalCountProvider` がこのドキュメントをリアルタイム購読して進捗バーに使用

### `users/{userId}/characters/{charId}/details/current`（既存ドキュメントへの追加フィールド）

```json
{
  "axisScores": {
    "energy":       0.4,
    "judgment":    -0.3,
    "relationship": 0.2,
    "lifestyle":    0.1,
    "processing":  -0.2
  },
  "element":    "炎",
  "typeName":   "場を沸かす炎タイプ",
  "convertedBig5Scores": {
    "openness": 2.8,
    "conscientiousness": 3.4,
    "extraversion": 3.8,
    "agreeableness": 3.6,
    "neuroticism": 2.1
  },
  "personalityNarrative": "最近、周りを巻き込む行動が目立っています",
  "axisUpdatedAt":      "Timestamp",
  "axisGeneratedAt":    "Timestamp",
  "narrativeUpdatedAt": "Timestamp"
}
```

- `axisGeneratedAt`: null なら signalCount 30到達時に `generateCharacterDetails` を実行・保存。再実行しないためのフラグ
- 既存フィールド（`confirmedBig5Scores`・`analysisLevel` 等）はそのまま残存（既存ユーザーの後方互換）

### `users/{userId}/characters/{charId}/big5Progress/current`（既存・後方互換）

旧BIG5 100問診断の進捗。新規ユーザーは使用しない。既存ユーザーのデータとして残存。

## `analysisLevel` フィールドの扱い

BIG5 100問診断廃止に伴い、`analysisLevel` フィールドは新システムでは使用しない。

| ユーザー種別 | 状態 |
|------------|------|
| 既存ユーザー | フィールドが存在するが参照しない（レガシーデータとして残す） |
| 新規ユーザー | BIG5質問がないためフィールドが生成されない（0のまま） |

Flutter の表示ロジックは `analysisLevel` への依存を除去し、代わりに `element` の有無で判定する。
