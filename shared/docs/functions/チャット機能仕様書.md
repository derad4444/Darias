## アーキテクチャ概要

```
ユーザー入力（ホーム画面）
    │
    ▼
ChatDatasource.sendMessageWithScheduleDetection()
    │
    ▼
classifyAndExtract（Cloud Function / gpt-4o-mini）
    │
    ├─① type: "memo"     → memos コレクションへ保存。複数件対応
    │
    ├─② type: "task"     → todos コレクションへ保存。複数件対応
    │
    ├─③ type: "schedule" → 確認ダイアログ → schedules コレクションへ保存
    │
    ├─④ type: "app_qa"   → answerAppQuestion（Cloud Function）
    │
    └─⑤ type: "chat"     → generateCharacterReply（Cloud Function）
            ├─ 直近2件のチャット履歴（posts）
            ├─ 直近30日の会議結論（meeting_history → shared_meetings）[Case 4]
            ├─ 悩み系キーワード検出 → meetingSuggested フラグ [Case 2]
            └→ posts コレクションに保存
```

## メッセージ振り分けロジック

振り分けは `classifyAndExtract` Cloud Function（AI）が行う。キーワードによる静的な判定は廃止。

### 分類の優先順位と判定基準

| 分類 | 判定基準 | 例 |
|------|---------|-----|
| `schedule` | 日時＋行動の組み合わせ | 「4/10に米米」「明日14時に会議」「来週月曜にジム」 |
| `memo` | メモ・記録の依頼 | 「メモして」「〜を記録して」 |
| `task` | タスク・やること登録の依頼 | 「タスクに追加」「ジョギングと買い物をTODOに」 |
| `app_qa` | アプリの使い方・機能に関する質問 | 「予定の編集方法は？」「タスクはどこで見れる？」 |
| `chat` | 上記以外の会話・相談・雑談 | すべての一般的な会話 |

**旧キーワードベース判定との主な違い:**
- 「4/10に米米」など日付パターンのみのメッセージも `schedule` として検出できる
- 「ジョギングと買い物をタスクに」など複数件も一度に抽出できる
- 誤検知（キーワードを含む雑談がメモ/タスク扱いになる等）が激減

## AIメッセージ分類

### Cloud Function: `classifyAndExtract`

メッセージを5種類に分類し、memo/task/scheduleの場合は内容も抽出して返す。1回のAI呼び出しで分類と抽出を同時実行（コスト効率）。

**モデル**: `gpt-4o-mini`（固定）

**入力:**
```javascript
{
  userMessage: string,
}
```

**返却（分類別）:**

```javascript
// memo（複数件対応）
{ "type": "memo", "items": ["メモ内容1", "メモ内容2"] }

// task（複数件対応）
{ "type": "task", "items": ["タスク内容1", "タスク内容2"] }

// schedule
{
  "type": "schedule",
  "schedules": [
    {
      "title": string,
      "isAllDay": boolean,
      "startDate": Timestamp,   // +09:00 付きISO文字列 → Firestoreタイムスタンプに変換
      "endDate": Timestamp,
      "location": string,
      "tag": string,
      "memo": string,
      "repeatOption": "none",
      "remindValue": 0,
      "remindUnit": "none",
      "created_at": Timestamp,
    }
  ]
}

// app_qa（内容抽出なし）
{ "type": "app_qa" }

// chat（内容抽出なし）
{ "type": "chat" }
```

**エラー時フォールバック**: `{ "type": "chat" }` を返してキャラクター返答を継続

**プロンプト**: `OPTIMIZED_PROMPTS.classifyAndExtract(currentDate, currentTime, userMessage)` を使用（`shared/functions/src/prompts/templates.js`）

### Flutter側のハンドリング

```dart
// classifyAndExtract 呼び出し
final callable = _functions.httpsCallable('classifyAndExtract');
final result = await callable.call({'userMessage': trimmed});
final type = result.data['type']; // "memo" | "task" | "schedule" | "app_qa" | "chat"
```

#### memo/taskの複数件対応

`items` 配列から全件 `MemoModel` / `TodoModel` に変換し、ユーザーが「追加」を押した際に全件Firestoreに保存。

```dart
// 複数件をダイアログに一覧表示
title: memos.length > 1 ? 'メモを${memos.length}件保存しますか？' : 'メモを保存しますか？'
// 全件保存
for (final m in memos) { await _saveMemoModel(m); }
```

- 1件の場合: 「メモを保存しますか？」 + [追加][編集][キャンセル]
- 複数件の場合: 「メモをX件保存しますか？」 + 全件リスト表示 + [全て追加][キャンセル]（編集ボタンなし）

#### scheduleの確認ダイアログ

→ `_ScheduleListConfirmDialog`（後述）

## 通常チャット

### Cloud Function: `generateCharacterReply`

**Cloud Function 側の処理の振り分け（優先順）:**

1. **無意味な入力検出**（3文字未満・記号のみ・繰り返し文字等）→ フォールバック固定返答
2. **予定問い合わせ**（「今日の予定」「明日の予定」等のパターン）→ Firestore `schedules` を照会して返答
3. **通常チャット** → OpenAI でキャラクター返答を生成

> **レガシーBIG5 100問診断（後方互換性のため維持）:**
> - BIG5トリガー（「性格診断して」「性格解析して」）→ 次の質問を返す
> - BIG5回答（1-5の数字 かつ `currentQuestion` が存在）→ 回答を記録して次の質問
>
> ※ 現在の主要な性格解析はチャットシグナル方式（`classifyAndExtract` が自動抽出するパーソナリティタグ）で行われる。BIG5 100問フローは既存ユーザーとの後方互換性のために維持されており、明示的にトリガーされた場合のみ実行される。

### 入力パラメータ

```javascript
{
  characterId: string,
  userMessage: string,      // 100文字まで処理
  userId: string,
  isPremium: boolean,
  chatHistory: [            // 直近2件の会話履歴
    { userMessage: string, aiResponse: string }
  ],
  meetingContext?: string,  // 直近30日の会議結論（後述 Case 4）
  phase: 1 | 2 | 3,        // 返答フェーズ（Flutter側で算出して送信、デフォルト: 1）
}
```

### 返却値

```javascript
{
  reply: string,
  voiceUrl: "",              // 常に空文字（音声は別途生成）
  // 感情表現（通常チャット時のみ値あり）:
  emotion?: string,          // "" (normal) / "_smile" / "_angry" / "_cry" / "_sleep"

  // ── レガシーBIG5 100問診断フィールド（後方互換性のため維持）──
  isBig5Question: boolean,   // BIG5回答モード中かどうか
  // BIG5回答時のみ:
  questionId?: string,
  questionText?: string,
  progress?: string,         // 例: "15/100"
  // BIG5完了時のみ:
  big5Completed?: boolean,
  newScores?: Big5Scores,
}
```

### モデル選択

| ユーザータイプ | モデル |
|-------------|--------|
| プレミアム | `gpt-4o-2024-11-20` |
| 無料 | `gpt-4o-mini` |

### キャラクタープロンプトの構成

`OPTIMIZED_PROMPTS.characterReply()` を使用。入力は以下:

- **type**: 性格タイプ（AI / Learning / Human）
- **gender**: male / female / neutral
- **big5**: 5つのスコア（openness, conscientiousness, extraversion, agreeableness, neuroticism）
  - `convertedBig5Scores`（チャットシグナルから変換）を優先し、なければ `confirmedBig5Scores`（旧BIG5 100問）を使用
- **dreamText**: キャラクターの夢
- **userMessage**: ユーザーのメッセージ
- **meetingContext**: 直近30日の会議結論（Case 4）

> **スコアの送り方**: `characterReply` は `buildPersonalityTraits()` で自然言語に変換して送信（数値は送らない）。`diary` / `activityDiary` / `characterDetails` は `formatBig5ShortWithTraits()` で数値＋短いラベルのコンパクト形式（例: `O3(バランス型)C4(計画的)E2(内向的)A5(共感型)N1(超安定)`）で送信。（2026-04-04 変更）

性格タイプは `androidScore` で決定:
```
androidScore = (6 - agreeableness) + (6 - extraversion) + (6 - neuroticism)

>= 9 → AI型    (論理的・システマティック)
<= 6 → 人間型  (共感・感情重視)
それ以外 → 学習型 (論理+感情のミックス)
```

### 返答プロンプトの構造 ― フェーズ別システム（2026-05-16 更新）

`characterReply` プロンプトは `phase` パラメータに応じて3段階の指示文を切り替える。  
性格データが不十分な段階（`signalCount < 30`）ではPhase 1固定とし、誤った性格言及を防ぐ。

#### フェーズ対応表

| フェーズ | 適用条件 | 文字数上限 | 返答スタイル | ユーザー体験 |
|---------|---------|----------|-------------|------------|
| **Phase 1** | signalCount < 30（全ターン）または signalCount ≥ 30 の session turn 1 | 120文字 | 受け止め＋読み＋質問（性格反映なし） | 自然な受け答え |
| **Phase 2** | signalCount ≥ 30 かつ session turn 2〜4 | 150文字 | 受け止め＋性格仮説＋質問 | 「あたってるかも？」体験 |
| **Phase 3** | signalCount ≥ 30 かつ session turn 5以上 | 220文字 | 受け止め＋性格言語化＋質問（任意） | 「なるほど、そういう人間なんだ」体験 |

#### フェーズ算出ロジック（Flutter: `home_screen.dart`）

```dart
int _calcPhase(int signalCount, int sessionTurn) {
  if (signalCount < 30) return 1;  // 元素未確定 → 性格反映なし
  if (sessionTurn <= 1) return 1;
  if (sessionTurn <= 4) return 2;
  return 3;
}
```

- `sessionChatCountProvider`：アプリ起動ごとにリセット（セッション単位で管理）
- `phase` は Flutter側で算出し、Cloud Function に渡す（サーバーは受け取ったphaseをそのまま使用）

#### Phase別プロンプト指示文

**Phase 1（120文字以内）:**
```
① ユーザーの言葉を1文で受け止める（共感・肯定）
② 読み取ったことを返す：「〜ってことかな」「〜が気になってるのかも」など（1文）
   ※答えを教えるのではなく、あなたの"読み"を提示する
③ 本音を引き出す問いを1つだけ
質問は1つ。答えを与えず、ユーザー自身が気づくよう促す。
```

**Phase 2（150文字以内）:**
```
① ユーザーの言葉を1文で受け止める
② 性格特性を踏まえた"仮説"を返す：「〜な面が出てる気がする」「あなたらしい選択だと思う」など（1〜2文）
   ※正解ではなく仮説として提示する。ズレていれば訂正させてよい
③ 仮説を確認するか深掘りする問いを1つ
質問は1つ。「あたってるかも？」と感じてもらえるような性格の洞察を心がける。
```

**Phase 3（220文字以内）:**
```
① 受け止め＋感じたことを1文で返す
② 性格特性に基づく深い洞察を2〜3文で：「〜という傾向があるから〜と感じるのは自然」「〜タイプはこういう場面でこう動きやすい」など
   ※性格タイプ名（炎タイプ等）を自然に使ってよい。ラベルを貼るのでなく"なぜそう感じるのか"を説明する
③ 自己理解を深める問いを1つ（任意：自然な流れなら省いてよい）
"なるほど、自分ってそういう人間なんだ"と納得させることを目指す。
```

#### `max_tokens` 設定

```javascript
// generateCharacterReply.js
max_tokens: phase === 2 ? 130 : phase === 3 ? 180 : 100
// Phase1: ~120文字 / Phase2: ~150文字 / Phase3: ~220文字
```

**設計意図:**
- signalCount < 30 は性格データが不十分なため Phase 1 固定（誤った性格言及を防ぐ）
- Phase 2/3 は元素タイプ確定後（signalCount ≥ 30）のみ有効
- Phase 3 ではtypeName（「炎タイプ」等）が返答中に自然に使われる
- Phase 2 は仮説提示→質問で終わる。Phase 3 は言語化が主で質問は任意

## アプリQ&A

### Cloud Function: `answerAppQuestion`

アプリの使い方に関する質問、またはユーザーデータ参照が必要な質問に回答。`classifyAndExtract` が `app_qa` と判定した場合に呼び出される。

**入力:**

```javascript
{
  userId: string,
  userMessage: string,
  dataTypes: string[],  // ["schedules", "todos", "memos"] から必要なもの
}
```

> `dataTypes` は Flutter 側でメッセージを解析して付与（予定・タスク・メモの参照が必要かを簡易判定）

**処理:**
1. `dataTypes` に基づき Firestore からデータ取得（読み取り専用）
   - `schedules`: 前後30日以内（最大20件）
   - `todos`: 未完了タスク（最大20件）
   - `memos`: 最新メモ（最大10件）
2. `APP_GUIDE`（アプリ全機能の説明）+ データをシステムプロンプトに組み込み
3. `gpt-4o-mini` で回答生成

**APP_GUIDE でカバーするトピック（`answerAppQuestion.js` 内に定義）:**
- チャット操作（メモ/タスク/予定の追加方法）
- メモ・タスク・予定の使い方（詳細操作含む）
- チャット履歴・日記の確認方法
- キャラクターについて（BIG5診断・変更方法）
- 日記の自動生成（アクティビティ型）
- 自分会議（6人会議）の使い方・利用制限
- フレンド機能（追加・申請・予定共有レベル）
- 相性診断（カテゴリ・解放方法・条件）
- プレミアム機能（高品質AI・自分会議月30回）
- タグ管理・設定（テーマ・音量・通知）
- アカウント管理（削除時の注意事項）

**返却:** `{ reply: string }` （postsに保存しない）

## 予定確認ダイアログ

### `_ScheduleListConfirmDialog`

抽出された予定を **カードリスト形式** で表示するStatefulWidget。1件でも複数件でも同じUIを使用。

```
┌──────────────────────────────────┐
│ 2件の予定を追加しますか？         │
│ ┌────────────────────────────┐   │
│ │ ランチミーティング  [編集] │   │
│ │ 4/5(火) 12:00〜13:00 [除外] │   │
│ │ 場所: 小会議①              │   │
│ └────────────────────────────┘   │
│ ┌────────────────────────────┐   │
│ │ 各社定例          [編集]   │   │
│ │ 4/7(木) 11:00〜12:00 [除外] │   │
│ └────────────────────────────┘   │
│                                   │
│ [キャンセル]     [2件追加する]    │
└──────────────────────────────────┘
```

カードレイアウト: 左側に予定情報（タイトル・日時・場所）、右側に[編集][除外]ボタンを縦積みで中央揃え。背景はアプリのテーマグラデーションを使用。

**操作:**
- **[編集]**: ダイアログはpopせずカレンダー詳細画面（`/calendar/detail`）へ `context.push<bool>` で遷移。ScheduleModelを `extra` に渡す。
  - 保存完了時: `schedule_detail_screen` が `context.pop(true)` を返す → ダイアログがawaitして受け取り、そのカードをリストから自動削除（二重登録防止）
  - キャンセル・バック時: `null` が返るのでカードはそのまま残る
- **[除外]**: `setState()` でリストから除去（即時反映）。全件除外でリスト空になると「全て除外されました」メッセージ＋[閉じる]ボタンに切り替わる
- **[X件追加する]**: `_saveScheduleModels(List<ScheduleModel>)` を呼び出し残件をFirestoreに保存。成功数を `SnackBar` で通知

**保存メッセージ:**
- 全件成功: 「X件の予定を追加しました」
- 一部成功: 「X/Y件の予定を追加しました」

Flutter側で `ScheduleModel` に変換し、`users/{userId}/schedules/{id}` へ保存。postsには保存しない。

**関連する型変更:**
- `onEdit: Future<bool?> Function(ScheduleModel)` — 保存済みかどうかをBoolで受け取る
- `schedule_detail_screen`: 保存成功時 `context.pop(true)`、削除・キャンセル時は `context.pop()` のまま

## 会議連携

### Case 1: 会議後フォローアップ（ホーム画面への固定メッセージ）

会議画面で「チャットで深掘りする」ボタンを押すと:

1. `meetingFollowupConclusionProvider`（`StateProvider<String?>`）に会議結論を書き込む
2. ホームタブ（index 0）に遷移
3. ホーム画面の `ref.listen<String?>` がプロバイダーの変化を検知
4. 固定メッセージ「会議お疲れ様！続きがあれば話しかけてね」を表示

**特徴:**
- Cloud Function の呼び出しなし（固定文）
- `IndexedStack` でホーム画面は常にマウント済みのため `ref.listen` が機能する

```dart
// meeting_screen.dart
ref.read(meetingFollowupConclusionProvider.notifier).state =
    _meetingResponse!.conversation.conclusion.summary;
ref.read(selectedTabProvider.notifier).state = 0;
context.go('/');

// home_screen.dart（build内）
ref.listen<String?>(meetingFollowupConclusionProvider, (prev, next) {
  if (next != null) {
    ref.read(meetingFollowupConclusionProvider.notifier).state = null;
    _triggerMeetingFollowup(next);
  }
});
```

### Case 2: 悩み系キーワードへの会議提案

通常チャット時に悩み系キーワードを検出すると `meetingSuggested: true` を返す。

**トリガーワード:**
```dart
'どうしよう', '迷ってる', '迷っている', '決められない', '悩んでいる',
'悩んでる', 'どうすればいい', 'どうしたらいい', '困ってる', '困っている',
'迷い', '悩み', 'どうしたら', '相談したい', '判断できない', 'わからなくなってきた'
```

Flutter側で `meetingSuggested == true` の場合に「6人会議を試す？」等のUIを表示（呼び出し元の実装依存）。

### Case 4: 会議結論をチャットのコンテキストに注入

通常チャット時、直近30日以内の会議結論を `generateCharacterReply` に `meetingContext` として渡す。

```dart
Future<String?> _fetchRecentMeetingConclusion(userId, characterId) async {
  // meeting_history から最新1件取得
  // 30日以内かチェック
  // shared_meetings から conclusion.summary を取得
  // "相談:{concern} / 結論:{summary(80文字)}" 形式で返す
}
```

Cloud Function 側ではこれをシステムプロンプトに組み込む。プロンプト形式:

```
【会議コンテキスト】相談:{concern} / 結論:{summary(80文字)}
（ユーザーが会議について話している場合はこの内容を踏まえて返答してください）
```

会議に関係する話題のときのみ言及するよう明示的に指示している。

## チャットオープナー

### 概要

ホーム画面を開いたとき（初回マウント時）に、ユーザーがチャットを始めるきっかけとなる「オープナーメッセージ」をキャラクターの吹き出しに表示する仕組み。AI呼び出しは行わない（クライアントサイドのみで完結）。

実装ファイル: `lib/presentation/screens/home/chat_opener.dart`

### 優先順位（G → F → A）

```
G: 今日の予定 or 期限切れ・今日期限の未完了タスクがある
    ↓（なければ）
F: 前回のAI返答から保存した問いかけ（未使用のもの）
    ↓（なければ or 使用済みなら）
A: デイリープロンプト（47種・日付ベースローテーション）
```

#### G: スケジュール/タスクオープナー

今日の予定または期限が今日以前の未完了タスクが存在する場合、タイトルを埋め込んだテンプレート文字列を表示。

```dart
// 予定テンプレート例
'今日「${s.title}」があるね。どんな気持ちで臨む？'
'${s.title}、今日だね。何か気になってることある？'

// タスクテンプレート例
'「${t.title}」の期限が近いね。今どんな状況？'
'${t.title}について、正直どう感じてる？'
```

テンプレートの選択は `today.day % templates.length` で決定（日によって変わるが同日中は固定）。

#### F: 前回の問い（1回限り）

AI返答の末尾から `？` で終わる最後の文を `saveLastQuestion()` で抽出・保存。次回起動時に1回だけ「前回の問い、その後どうなった？」として再提示し、表示後は使用済みフラグを立てる。

**SharedPreferences キー:**

| キー | 型 | 用途 |
|-----|-----|------|
| `chat_last_question` | `String` | 前回の問いかけ文 |
| `chat_question_used` | `bool` | 使用済みフラグ（true = 表示済み） |
| `chat_opener_saved_date` | `String` | オープナーをFirestoreに保存した最終日付（`YYYY-MM-DD`形式）。同日内の重複保存を防ぐ |

```dart
// 表示時
await prefs.setBool(_sharedPrefKeyUsed, true);
return ChatOpener(
  type: OpenerType.previousQuestion,
  text: '前回の問い、その後どうなった？\n「$lastQuestion」',
);
```

#### A: デイリープロンプト

47種類の問いかけが `dailyPrompts` リストにハードコード。

```dart
final dayIndex = today.difference(DateTime(2024, 1, 1)).inDays.abs();
final prompt = dailyPrompts[dayIndex % dailyPrompts.length];
```

1日中同じプロンプトを表示（同日内は不変）。カテゴリは以下:

| カテゴリ | 件数 |
|---------|------|
| 朝・スタート系 | 7 |
| 振り返り系 | 6 |
| 自己探求系 | 11 |
| 思考整理系 | 7 |
| 軽め・雑談系 | 6 |
| 深掘り系 | 7 |

### `saveLastQuestion()` の呼び出しタイミング

AI返答が返るたびに末尾の問いかけ文を保存する。

```dart
// _sendMessage() 内
if (result != null) saveLastQuestion(result.reply);

// _triggerMeetingFollowup() 内
if (result != null) saveLastQuestion(result.reply);
```

`saveLastQuestion()` は返答を `。！\n` で分割し、`？` または `?` で終わる最後の文を抽出。抽出できた場合のみ `chat_question_used: false` で保存（次回Fオープナーとして使用可能になる）。

### ロード処理・Firestore保存

```dart
// home_screen.dart initState()
WidgetsBinding.instance.addPostFrameCallback((_) => _loadChatOpener());

Future<void> _loadChatOpener() async {
  if (!mounted || _openerLoaded) return;
  _openerLoaded = true;  // 再実行防止
  final schedules = ref.read(allSchedulesProvider).valueOrNull ?? [];
  final todos = ref.read(todosProvider).valueOrNull ?? [];
  final opener = await computeChatOpener(allSchedules: schedules, allTodos: todos);
  if (mounted) {
    setState(() {
      _displayedMessage = opener.text;
      _isCharacterReply = true;
    });
    await _saveOpenerIfNeeded(opener.text);  // 1日1回Firestoreに保存
  }
}
```

オープナーは `_saveOpenerIfNeeded()` で1日1回だけ `posts` コレクションに保存される（`content: ""`, `analysis_result: opener.text`）。`chat_opener_saved_date` と当日日付が一致する場合はスキップする。

初期表示時は時間帯挨拶/ランダムメッセージを暫定表示し、プロバイダーのデータがロードされてから上書きする。

## 性格解析システムとの連携

### チャットシグナルによる軸スコア・元素タイプ解析（主要システム）

チャットメッセージを `classifyAndExtract` が分類する際、メッセージの特徴を示す**パーソナリティタグ**（最大3件）も同時に抽出する。有効なタグが含まれる場合は `personalitySignals` に保存し、累積カウントが **10の倍数**になるたびに `calculateAndSaveAxisScores` が実行されて `details/current` の軸スコア・元素タイプを更新する。

**有効なパーソナリティタグ（16種）:**
```
social_reference, solo_preference, group_activity, initiating,
quiet_environment, planning_language, spontaneous_language, goal_oriented,
emotional_expression, logical_reasoning, intuitive_decision, data_driven,
cooperative_language, independent_stance, change_seeking, worry_anxiety
```

**軸スコア計算:**
- 過去90日・最大200件のシグナルを使用（時間減衰: 0.95^日数）
- 5軸スコア（energy / lifestyle / relationship / processing / judgment）を -1.0〜+1.0 で算出
- 軸スコアから元素（炎/風/雷/光/水/土/氷/闇/無）と typeName を決定
- `details/current` に `axisScores`, `element`, `typeName`, `convertedBig5Scores`, `axisUpdatedAt` を書き込む

**`_Big5AnalysisSection`（✨人格解析欄）の表示条件:**

```dart
if (signalCount >= 30 && detail.element != null)
  // _Big5AnalysisSection を表示
```

- `element` は10シグナル時点でも `calculateAndSaveAxisScores` によって書き込まれる
- しかし `generateCharacterDetails`（実際の詳細解析テキスト生成）は **30シグナル到達時に1回だけ**実行される（`axisGeneratedAt` がない場合のみ）
- 10シグナル時点では `element` が存在しても詳細解析データが未生成のため、ローディング状態が続く
- このためキャラクター詳細画面の表示しきい値を `signalCount >= 30` としている

### レガシーBIG5 100問診断フロー（後方互換性のため維持）

> **注意**: このフローは既存ユーザーとの後方互換性のために維持されています。新規ユーザーはチャットシグナル方式（上記）により自動で性格解析されます。

```
ユーザー: 「性格診断して」
    ↓
generateCharacterReply（Cloud Function）
    ├─ isTopicRequest: true を検出
    ├─ big5Progress/current から次の質問を取得
    └─ 質問文を返却（isBig5Question: true）

ユーザー: 「3」（1-5の数字）
    ↓
generateCharacterReply
    ├─ isNumericAnswer: true かつ currentQuestion が存在
    ├─ 回答を answeredQuestions に追加
    ├─ 段階完了チェック（20問、50問、100問）
    │   └─ 段階完了時: generateStagedCharacterDetails を非同期実行
    └─ 次の質問または完了メッセージを返却
```

**BIG5 回答時の返却仕様:**
- 回答処理中はOpenAI APIを一切呼び出さない
- `reply: ""` （空文字）、`emotion: ""` （空文字）を返す
- 100問完了時のみ `getStageCompletionMessage(3)` の固定テキストを `reply` に設定
- 選択肢テキストは含めず、`questionText` フィールドに質問文のみを返す

**診断リセット（性格リセット）:**

リセット起点: 設定画面「性格診断をリセット」ボタン / BIG5診断画面「リセット」ボタン（`big5_datasource.dart: resetDiagnosis()`）

| 対象 | 操作 |
|------|------|
| `users/{userId}/characters/{characterId}/big5Progress/current` | ドキュメント削除 |
| `users/{userId}/characters/{characterId}/details/current` | `confirmedBig5Scores`, `personalityKey`, `analysis_level`, `sixPersonalities`, キャラクター属性15項目, `axisScores`, `element`, `typeName`, `convertedBig5Scores`, `axisUpdatedAt`, `axisGeneratedAt`, `personalityNarrative` を削除 |
| `users/{userId}/characters/{characterId}/generationStatus/current` | ドキュメント削除 |
| `users/{userId}/personalityMeta/current` | ドキュメント削除（signalCount=0相当） |
| `users/{userId}/personalitySignals/{*}` | サブコレクション内全ドキュメント削除 |
| `users/{userId}.growthStage` | `0` にリセット |
| `sessionChatCountProvider`（Flutter in-memory） | `0` にリセット（フェーズ算出のセッションターン数） |

## Firestoreへの保存

### postsに保存されるケース

**保存される**:
- 通常チャット（`generateCharacterReply` の5番ルート）→ `content` + `analysis_result` 両方あり
- チャットオープナー（1日1回）→ `content: ""` + `analysis_result: オープナー文字列`

**保存されない**: 以下のケース
- アプリQ&A（answerAppQuestion）
- memo（memosコレクションに保存）
- task（todosコレクションに保存）
- schedule（schedulesコレクションに保存）
- BIG5回答・トリガー

### 保存データ

```
users/{userId}/characters/{characterId}/posts/{docId}
├── content: string        // ユーザーメッセージ（最大100文字）。オープナー保存時は ""
├── analysis_result: string // AI返答、またはオープナーテキスト
└── timestamp: Timestamp
```

**書き込み場所**: Cloud Function ではなく Flutter クライアントが保存する。
- 通常チャット: `ChatDatasource._savePost()`
- オープナー: `ChatDatasource.saveOpenerPost()`

### 履歴画面での表示

`unified_history_screen.dart` の `_ChatHistoryTab` でレンダリング。

- `content` が空のドキュメント（オープナー）はユーザーバブルを表示せず、`analysis_result` のキャラクターバブルのみ表示する
- `analysis_result` が空の場合もキャラクターバブルを表示しない（`if (message.analysisResult.isNotEmpty)` チェック済み）

## 画面構成

チャット機能はホーム画面（`home_screen.dart`）に内包されている。独立した `/chat` 画面は削除済み。

```
MainShellScreen（IndexedStack・5タブ）
├── タブ0: HomeScreen（常にマウント）
│   ├── キャラクター吹き出し（_displayedMessage）
│   │   └── 初期表示: computeChatOpener() によるスマートオープナー
│   ├── チャット入力フィールド
│   └── 送信ボタン
│       └─ onSend → ChatController.sendMessage() → ChatDatasource
│           └─ AI返答後: saveLastQuestion(result.reply)
├── タブ1: PlanScreen（手帳）
│   └── セグメント切替: 予定（CalendarScreen）/ タスク / メモ
├── タブ2: CharacterDetailScreen（詳細）
├── タブ3: FriendScreen（フレンド）
└── タブ4: SettingsScreen（設定）
```
