# チャットからのデータ入力機能 設計書

> ホーム画面のチャット入力から、自然言語でメモ・タスク・予定の追加、アプリQ&A、ユーザーデータ参照を行う機能の設計。

**最終更新日**: 2026-04-17（AI ベース分類方式へ移行。旧キーワードベース方式の記述を削除）
**対象ファイル**:
- `flutter/lib/data/datasources/remote/chat_datasource.dart`
- `flutter/lib/presentation/screens/home/home_screen.dart`
- `shared/functions/const/classifyAndExtract.js`
- `shared/functions/const/answerAppQuestion.js`

---

## 概要

チャット入力欄にメッセージを送ると、**`classifyAndExtract` Cloud Function（AI）** がメッセージを5種類（memo/task/schedule/app_qa/chat）に分類し、内容に応じて処理を振り分ける。

> **重要**: 2026-04-04 以前はクライアント側でキーワードによる静的な判定（contains）を行っていたが、「4/10に米米」など日付パターンのみのメッセージを schedule として検出できない・誤検知が多いといった問題があったため、AI ベースの `classifyAndExtract` に全面移行した。旧キーワードベースの実装は削除済み。

通常のキャラクター返答（`posts` 保存）とは独立して動作し、検出されたデータは **ユーザーの確認ダイアログを経てから** Firestore に保存される（Q&Aは即時返答）。

---

## 処理フロー（現行 AI ベース）

```
ユーザーがメッセージ送信
    ↓
[Cloud Function: classifyAndExtract]（gpt-4o-mini、毎回実行）
    │  分類結果 type: "memo" / "task" / "schedule" / "app_qa" / "chat"
    │
    ├─ "memo"     → 確認ダイアログ → memos コレクションへ保存（複数件対応）
    │
    ├─ "task"     → 確認ダイアログ → todos コレクションへ保存（複数件対応）
    │
    ├─ "schedule" → _ScheduleListConfirmDialog → schedules コレクションへ保存
    │
    ├─ "app_qa"   → [Cloud Function: answerAppQuestion] → 即時返答（posts 保存なし）
    │
    └─ "chat"     → [Cloud Function: generateCharacterReply] → posts に保存
```

> **注意**: memo・task・schedule・app_qa として処理されたメッセージは `posts` コレクションに保存されない。アクティビティ型日記（scheduledDiaryGeneration）はそれぞれのコレクションを直接参照するため日記には反映される。

## 分類の優先順位と判定基準（classifyAndExtract）

| 分類 | 判定基準 | 例 |
|------|---------|-----|
| `schedule` | 日時＋行動の組み合わせ | 「4/10に米米」「明日14時に会議」「来週月曜にジム」 |
| `memo` | メモ・記録の依頼 | 「メモして」「〇〇を記録して」 |
| `task` | タスク・やること登録の依頼 | 「タスクに追加」「ジョギングと買い物をTODOに」 |
| `app_qa` | アプリの使い方・機能に関する質問 | 「予定の編集方法は？」「タスクはどこで見れる？」 |
| `chat` | 上記以外の会話・相談・雑談 | すべての一般的な会話 |

旧方式との主な違い:
- 「4/10に米米」など日付パターンのみのメッセージも `schedule` として検出できる
- 「ジョギングと買い物をタスクに」など複数件を一度に抽出できる
- 誤検知（キーワードを含む雑談がメモ/タスク扱いになる等）が激減

**エラー時フォールバック**: `{ "type": "chat" }` を返してキャラクター返答を継続

## 1. アプリQ&A / ユーザーデータ参照

classifyAndExtract が `app_qa` と判定した場合に呼び出される。

### 処理フロー

```
classifyAndExtract → "app_qa"
    ↓
[Cloud Function: answerAppQuestion] 呼出
    ↓ APP_GUIDE（機能説明） + 必要に応じて Firestore からユーザーデータを取得
    ↓ gpt-4o-mini で回答生成（100文字以内）
    ↓
チャット返答として即時表示（確認ダイアログなし・posts 保存なし）
```

### ユーザーデータ参照

`answerAppQuestion` は以下の `dataTypes` で Firestore データを取得して回答に使用する。

| dataTypes 値 | 取得するデータ | Firestoreパス |
|-------------|-------------|-------------|
| `"schedules"` | 前後30日の予定（最大20件） | `users/{uid}/schedules` |
| `"todos"` | 未完了タスク（最大20件） | `users/{uid}/todos` |
| `"memos"` | 最新メモ（最大10件） | `users/{uid}/memos` |

> Flutter 側でメッセージを解析して `dataTypes` を付与する（予定・タスク・メモ等のキーワードを含む場合は対応するデータを取得）

### APP_GUIDE（answerAppQuestion 内の知識ベース）

`answerAppQuestion.js` 内の `APP_GUIDE` 定数として埋め込まれたアプリ機能説明。AI はこれを参照して回答する。以下の機能が網羅されている:

| カテゴリ | 主な内容 |
|---------|---------|
| チャットでできること | メモ/タスク/予定の追加・キャラクター会話・Q&A |
| メモの使い方 | チャット追加・メモ画面追加・タグ・メモからのタスク登録 |
| タスクの使い方 | チャット追加・完了・優先度・期限設定 |
| 予定の使い方 | チャット追加・繰り返し・リマインダー・フレンドの予定確認 |
| チャット履歴・日記 | 履歴ボタン・タブ切替・検索・会議結論の自動反映 |
| キャラクターについて | BIG5診断・診断の始め方・結果確認・キャラクター変更 |
| 日記（自動生成） | 生成タイミング・アクティビティ型・コメント追記 |
| 自分会議（6人会議） | 6分身・利用制限（無料1回/プレミアム月30回）・会議結論活用 |
| フレンド機能 | フレンド追加・申請承認・予定共有レベル設定・フレンドの予定表示 |
| 相性診断 | カテゴリ・スコア・解放方法（リワード広告）・診断条件 |
| プレミアム機能 | 高品質AIモデル・自分会議月30回・加入/解約方法 |
| タグ管理 | タグ追加・色設定・公開設定 |
| 設定 | テーマ・音量・通知・データエクスポート |
| アカウント管理 | パスワード変更・アカウント削除の注意事項 |

### answerAppQuestion Cloud Function

| 項目 | 値 |
|-----|-----|
| **関数名** | `answerAppQuestion` |
| **モデル** | `gpt-4o-mini` |
| **リージョン** | `asia-northeast1` |
| **メモリ** | 256MiB |
| **タイムアウト** | 60秒 |

**入力:**

| パラメータ | 型 | 説明 |
|-----------|-----|------|
| `userId` | `string` | Firebase Auth UID |
| `userMessage` | `string` | ユーザーのメッセージ |
| `dataTypes` | `array<string>` | 取得するデータ種別（`"schedules"` / `"todos"` / `"memos"`） |

**出力:**

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `reply` | `string` | 回答テキスト（100文字以内） |

> **注意**: Cloud Function はデータ参照のみ行い、Firestore への書き込みは行わない。

## 2. メモ追加

classifyAndExtract が `memo` と判定した場合。

### 処理フロー

```
classifyAndExtract → "memo"
    返却値: { type: "memo", items: ["メモ内容1", "メモ内容2"] }
    ↓
確認ダイアログ表示
    ・1件: 「メモを保存しますか？」[追加][編集][キャンセル]
    ・複数件: 「メモをX件保存しますか？」全件リスト表示 [全て追加][キャンセル]
    ↓ 追加 → users/{userId}/memos/{autoId} に保存
    ↓ 編集 → /memo/detail 画面へ遷移（単件のみ）
    ↓ キャンセル → 破棄
```

### 保存先

```
users/{userId}/memos/{autoId}
```

| フィールド | 値 |
|-----------|-----|
| `title` | classifyAndExtract が抽出したメモ内容 |
| `content` | `""` |
| `tag` | `""` |
| `isPinned` | `false` |
| `createdAt` | 現在日時 |
| `updatedAt` | 現在日時 |

### チャット返答

`"メモしておくね！"`（固定文）

## 4. 予定追加

classifyAndExtract が `schedule` と判定した場合。

### 処理フロー

```
classifyAndExtract → "schedule"
    返却値: { type: "schedule", schedules: [...] }（複数件対応）
    ↓
_ScheduleListConfirmDialog 表示（カードリスト形式）
    ┌─ 各カード: [編集] [除外] ボタン付き
    ├─ [編集]: /calendar/detail に push → 保存完了でカードをリストから自動削除
    ├─ [除外]: そのカードをリストから除去（全件除外で「全て除外されました」メッセージ）
    └─ [X件追加する]: 残件をFirestoreに保存 → SnackBarで成功件数を通知
```

### 返却データ形式（classifyAndExtract から）

```javascript
{
  "type": "schedule",
  "schedules": [
    {
      "title": string,
      "isAllDay": boolean,
      "startDate": Timestamp,   // +09:00 付き ISO → Firestore Timestamp 変換済み
      "endDate": Timestamp,
      "location": string,
      "tag": string,
      "memo": string,
      "repeatOption": "none",
      "remindValue": 0,
      "remindUnit": "none",
      "created_at": Timestamp
    }
  ]
}
```

### 保存先

```
users/{userId}/schedules/{autoId}
```

スキーマ詳細は `firebase/Firestoreスキーマ.md` の `users/{userId}/schedules` セクション参照。

### チャット返答

`"予定楽しんでね！"`（固定文）

## 予定確認ダイアログ（_ScheduleListConfirmDialog）

```
┌──────────────────────────────────┐
│ 2件の予定を追加しますか？         │
│ ┌────────────────────────────┐   │
│ │ ランチミーティング   [編集] │   │
│ │ 4/5(火) 12:00〜13:00 [除外] │   │
│ │ 場所: 小会議①              │   │
│ └────────────────────────────┘   │
│ ┌────────────────────────────┐   │
│ │ 各社定例           [編集]  │   │
│ │ 4/7(木) 11:00〜12:00 [除外] │   │
│ └────────────────────────────┘   │
│                                   │
│ [キャンセル]     [2件追加する]    │
└──────────────────────────────────┘
```

- [編集]: `/calendar/detail` へ遷移、保存完了時にそのカードをリストから自動削除（二重登録防止）
- [除外]: 即時リストから除去。全件除外で「全て除外されました」メッセージに切替
- [X件追加する]: 残件を保存、成功/部分成功を SnackBar で通知
- ダイアログは `barrierDismissible: false`（外タップで閉じない）

## 制約・注意事項

- メッセージの最大長は **100文字**（クライアントUI・generateCharacterReply Cloud Function 双方で制限）
- `classifyAndExtract` は全メッセージに対して呼び出されるため、1メッセージあたり約¥0.017 のコストが常にかかる
- 誤分類は `{ "type": "chat" }` にフォールバックされるため、最悪の場合でもキャラクター返答が返る
- ダイアログでキャンセルしてもキャラクターの返答（固定文）は既に表示済み
