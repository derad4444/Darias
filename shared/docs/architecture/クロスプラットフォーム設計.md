### カレンダー・タグ UI詳細

#### カレンダー画面（calendar_screen.dart）
- **ヘッダー**: `< 2026年 3月 >` 左右矢印で月移動、中央タップで年月ピッカーシート表示
- **「今日」ボタン**: ヘッダー右側、タップで今月・今日に戻る
- **年月ピッカーシート**: ボトムシート（全画面展開）、年・月スクロールホイール、ダブルタップで自由入力、「決定」ボタンで確定

#### 予定追加・編集画面（schedule_detail_screen.dart）
- **日付選択**: インライン展開式カレンダー（`CalendarDatePicker`ヘッダー非表示）
- **年月セレクター**: `< 2026年3月 ▼ >` 左右矢印で月移動、中央タップでスクロールホイール展開、ダブルタップで自由入力（年: 4桁制限+範囲クランプ、月: 1-12制限）
- **時間選択**: スクロールホイール（`ListWheelScrollView`）、タップで自由入力（時: 0-23、分: 0-59、`_MaxValueFormatter`で制限）
- **タグ選択**: Firestoreタグコレクション連携、カラーサークル付き表示
- **繰り返し設定**: ボトムシート形式、RepeatType（なし/毎日/毎週/毎月/月初/月末）、終了条件（なし/日付指定/回数指定）、終了日インラインカレンダー（今年〜20年先）、過去日付警告+完了ボタン無効化、プレビュー表示
- **通知設定**: ON/OFFトグル（通知実行制御のみ）、時間設定は常に表示、+/-ステッパー+ピルボタン（分/時間/日）

#### タグ設定画面（tag_management_screen.dart）
- **タグ一覧**: カラーバー付きカード、タグ名+メモ表示（最大2行省略）
- **タグ編集シート**: タグ名入力、カラーピッカー（プリセット30色）、メモ入力（複数行、任意）
- **Firestoreスキーマ**: `users/{userId}/tags/{tagId}` - name, colorHex, memo

## 6. プラットフォーム別の注意点

### iOS
- App Store 審査ガイドラインに準拠
- App Tracking Transparency (ATT) 対応
- StoreKit 2 から in_app_purchase への移行
- **ホーム画面ウィジェット**: WidgetKit (SwiftUI) で実装
  - Extension: `DariasWidgets` ターゲット（`ios/DariasWidgets/`）
  - App Group: `group.com.derad.Character` でFlutterアプリと共有UserDefaultsを通じてデータ連携
  - Flutter側: `home_widget` パッケージ（`WidgetDataService`）がJSON文字列をApp Groupに書き込み
  - Swift側: `WidgetDataCache` が読み取り・デコード（日付はString型で保持しJSONデコードエラーを回避）
  - ウィジェット種別: カレンダー予定リスト・月間カレンダーグリッド・メモ・Todo（各 Small/Medium/Large 対応）
  - カラー定義: `WidgetColors.swift` でアプリのピンク (#FF6B9D) /ラベンダー (#C44AC0) 配色を統一
  - **詳細設計**: `docs/architecture/ホーム画面ウィジェット設計.md` を参照
  - **Xcodeビルド注意**: 必ず `Runner.xcworkspace` を開くこと（`Runner.xcodeproj` では Pod フレームワーク未解決でビルドエラー）

### Android
- Google Play 審査ガイドラインに準拠
- Google Play Billing Library 対応
- 通知チャンネル設定
- バックグラウンド制限対応
- **ホーム画面ウィジェット**: `AppWidgetProvider` (Kotlin) で実装
  - `android/app/src/main/kotlin/com/darias/darias/` 配下に各ウィジェットProvider
  - レイアウト: `android/app/src/main/res/layout/` 配下のXMLで定義（Small/Large + 週数別バリアント対応）
  - データ連携: `home_widget` パッケージ経由でSharedPreferencesに書き込み（キーはiOSと共通）
  - ウィジェット種別: カレンダー予定リスト・月間カレンダーグリッド（週数別レイアウト w0〜w5）・メモ・Todo
  - **詳細設計**: `docs/architecture/ホーム画面ウィジェット設計.md` を参照

### Web
- Firebase Hosting でデプロイ
- PWA対応 (Service Worker)
- Stripe決済 (アプリ内課金の代替)
- 広告: Google AdSense に変更
- レスポンシブデザイン対応

```
┌─────────────────────────────────────────────────────────────────┐
│                      課金フロー                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │

│   ┌─────────────┐                ┌─────────────┐               │

│   │    SDK      │                │  Checkout   │               │

│          │                              │                       │
│          │    ┌───────────────────┐     │                       │
│          └────►  Firebase         ◄─────┘                       │
│               │  Cloud Functions  │                             │
│               │  (Webhook処理)    │                             │

│                         ▼                                       │

### RevenueCat を推奨する理由
- iOS/Android の課金を統一API で管理
- サーバーサイド検証が自動
- ダッシュボードで売上分析
- Firestore との連携が容易

name: Flutter CI/CD

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: subosito/flutter-action@v2
      - run: flutter pub get
      - run: flutter test
      - run: flutter analyze

  build-android:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: subosito/flutter-action@v2
      - run: flutter build apk --release

  build-ios:
    needs: test
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      - uses: subosito/flutter-action@v2
      - run: flutter build ios --release --no-codesign

  build-web:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: subosito/flutter-action@v2
      - run: flutter build web --release
      - uses: FirebaseExtended/action-hosting-deploy@v0
```

---

## 11. アーキテクチャの利点

1. **単一コードベース**: 1回の改修で iOS/Android/Web に反映
2. **DB共通**: FirestoreをiOS/Android/Web共通で使用
3. **バックエンド共通**: Cloud Functionsを全プラットフォームで流用
4. **保守性向上**: 今後の機能追加が1箇所で完結
