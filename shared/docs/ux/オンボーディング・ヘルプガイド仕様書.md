## 概要

新規ユーザーへの機能説明と、既存ユーザーへの操作ヒント表示を担う機能群。

| 機能 | 表示タイミング | 管理場所 |
|------|--------------|---------|
| オンボーディングスライド | 新規登録後・未視聴アカウントのログイン時 | Firestore `hasSeenOnboardingSlides` |
| 使い方ガイド画面 | 設定画面から任意で参照 | ハードコード |
| インラインヒントバナー | 各画面への初回アクセス時のみ | SharedPreferences（端末ローカル） |
| 相性診断 性格解析チェック | 相性診断画面（常時） | Firestoreリアルタイム |

## オンボーディングスライド

### 表示フロー

```
新規登録
    ↓
register_screen.dart
    ├── needsOnboardingProvider = true（signUp() 呼び出し前にセット）
    └── signUp() 完了
            ↓
        GoRouter redirect（refreshListenable でトリガー）
            ├── isLoggedIn && isAuthRoute の場合
            ├── needsOnboardingProvider == true → /onboarding へ
            └── Future.microtask でフラグをリセット

ログイン（splash経由）
    ↓
Firestore users/{uid}.hasSeenOnboardingSlides を確認
    ├── false または フィールドなし → /onboarding へ遷移
    └── true → / (ホーム) へ遷移
```

> **実装注意**: GoRouter の `redirect` はログイン完了を検知して即時発火するため、`context.go('/onboarding')` を `signUp()` 完了後に呼んでも redirect が先に `/` へ飛ばしてしまう。そのため `needsOnboardingProvider`（`StateProvider<bool>`）を事前にセットし、redirect 内で参照する方式を採用している。エラー時はフラグをリセット（`needsOnboardingProvider = false`）する。

### スライド構成（4枚）

| # | タイトル | アイコン | 内容 |
|---|---------|---------|------|
| 1 | DARIASへようこそ | `auto_awesome` | アプリ概要 |
| 2 | チャットで何でもできる | `chat_bubble_outline` | チャット操作・Q&A・使い方ガイドの案内 |
| 3 | 3つのAI機能 | `psychology_outlined` | 日記・自分会議・相性診断の説明 |
| 4 | 性格解析について | `star_outline` | チャット自動解析の説明・使い方ガイドへの誘導 |

#### スライド3「3つのAI機能」本文

```
📔 日記：1日の内容を毎晩自動生成

👥 自分会議：6つの自分が悩みをディスカッション

💞 相性診断：フレンドとの性格相性をチャット解析で診断
```

#### スライド4「性格解析について」本文

```
普通にチャットするだけで、AIが自動的に
あなたの性格を解析していきます。

30回以上チャットすると性格タイプが判定され、
キャラクターの返答・日記・会議・相性診断に反映されます。

設定 → 使い方ガイドからいつでも確認できます。
```

> **旧仕様からの変更（2026-05-10）**: スライド4は「まず性格診断から（BIG5 100問回答）」という内容だったが、チャット自動解析システム（性格診断リニューアル）の導入に伴い、チャットを続けることで性格が自動解析される旨に変更した。

### 操作

| ボタン | 動作 |
|--------|------|
| スキップ | Firestoreに`hasSeenOnboardingSlides: true`を書き込み → ホームへ |
| 次へ | 次のスライドへ |
| 始める！（最終ページ） | Firestoreに`hasSeenOnboardingSlides: true`を書き込み → ホームへ |

### Firestore管理

- **フィールド**: `users/{userId}.hasSeenOnboardingSlides` (`boolean`)
- **設定タイミング**: スキップ or 始める！ボタン押下時
- **初期値**: フィールドなし（= `false` 扱い）
- **目的**: 端末・ブラウザ問わずアカウント単位での表示制御

> **注意**: 既存フィールド `hasCompletedOnboarding` はキャラクター作成完了フラグとして独立して使用。オンボーディングスライドの表示制御には `hasSeenOnboardingSlides` を使用する。

## インラインヒントバナー

### 概要

各画面への**初回アクセス時のみ**表示される説明バナー。  
アニメーション（`SizeTransition`）付きで表示され、×ボタンで閉じると以降は表示されない。

### 管理方法（SharedPreferences）

キー形式: `hint_v1_{feature}_{userId}`

アカウント切り替えには対応しているが、**端末・ブラウザをまたいだ同期はしない**（ローカル保存）。  
→ 別端末では同じアカウントでも再表示される可能性あり（軽微な仕様として許容）

### 定義済み feature キー（`HintService`）

| 定数 | キー | 設置画面 |
|------|-----|---------|
| `kHome` | `home` | ホーム画面（3ステップバナー） |
| `kMemo` | `memo` | メモ編集画面 |
| `kMeeting` | `meeting` | 自分会議画面（初回に参加者説明ダイアログを自動表示） |
| `kFriend` | `friend` | フレンド一覧画面 |
| `kCompatibility` | `compatibility` | 相性診断画面 |
| `kCalendarDiary` | `calendar_diary` | カレンダー予定一覧（日記アイコン説明） |

### バナー種別

#### `InlineHintBanner`（汎用）

- 1メッセージを表示
- 右端の×ボタンで閉じる → `HintService.markShown()` を呼び出し

```dart
InlineHintBanner(
  userId: userId,
  feature: HintService.kMemo,
  message: 'ツールバーの「タスク」ボタンを押すと...',
  icon: Icons.check_box_outlined,
)
```

#### `HomeHintBanner`（ホーム専用）

- 3ステップをタップで順送り（「1/3 ›」カウンター表示）
- 全ステップ完了でバナーが閉じる
- ステップ進捗は `HintService.getHomeStep()` / `setHomeStep()` で管理

| Step | アイコン | 内容 |
|------|---------|------|
| 1 | `chat_bubble_outline` | チャットで予定・メモ・タスク登録 |
| 2 | `help_outline` | チャットQ&A + 設定から使い方ガイド |
| 3 | `psychology_outlined` | 性格解析の案内（チャットで自動解析） |

### 設置一覧

| 画面 | ウィジェット種別 | 設置箇所 | 内容 |
|------|--------------|---------|------|
| ホーム画面 | `HomeHintBanner` | Stack上部（Positioned top:0） | 3ステップ機能紹介 |
| メモ編集 | `InlineHintBanner` | エディタの上 | タスクボタンの説明 |
| 自分会議 | ダイアログ自動表示 | initState（初回のみ） | 参加キャラクター説明 |
| フレンド一覧 | `InlineHintBanner` | フレンド一覧の上 | 共有レベルの説明 |
| 相性診断 | `InlineHintBanner` | アバター行の下 | 広告・性格解析必要条件の説明 |
| カレンダー予定一覧 | `InlineHintBanner` | 日記アイコンの上 | 日記アイコンの説明 |

### アカウント削除時のクリーンアップ

`HintService.clearAllForUser(userId)` を呼び出し、  
`_{userId}` で終わるすべてのSharedPreferencesキーを削除。  
→ `settings_screen.dart` の `_confirmDeleteAccount` 内で実行。

### 実装ファイル

- `lib/data/services/hint_service.dart`
- `lib/presentation/widgets/inline_hint_banner.dart`

## 相性診断 性格解析チェック

### 概要

相性診断はフレンドと自分双方の性格解析が必要。  
性格解析は `confirmedBig5Scores`（旧BIG5 100問方式、後方互換）または `convertedBig5Scores`（チャット自動解析の5軸変換済みスコア）のいずれかが存在すれば利用可能。  
どちらも存在しない場合は「診断する」ボタンを非活性にし、理由を画面に表示する。

> **旧BIG5方式について**: `confirmedBig5Scores` はBIG5 100問診断を完了したユーザーのスコアで、既存ユーザーの後方互換として残存する。新規ユーザーはチャット自動解析（30回以上）により `convertedBig5Scores` が生成される。

### 判定ロジック

```dart
// 自分の性格解析スコア（confirmedBig5Scores OR convertedBig5Scores）
final myDetails = ref.watch(characterDetailsProvider).valueOrNull;
final myBig5Done = (myDetails?.confirmedBig5Scores?.isNotEmpty ?? false)
    || (myDetails?.convertedBig5Scores?.isNotEmpty ?? false);

// フレンドの性格解析スコア
final friendDetailsAsync = ref.watch(userCharacterDetailsProvider(friend.id));
final friendBig5Done =
    (friendDetailsAsync.valueOrNull?.confirmedBig5Scores?.isNotEmpty ?? false)
    || (friendDetailsAsync.valueOrNull?.convertedBig5Scores?.isNotEmpty ?? false);

// ロード中は誤表示を防ぐため活性にしておく
final dataLoaded = !characterDetailsProvider.isLoading && !friendDetailsAsync.isLoading;
final ready = !dataLoaded || (myBig5Done && friendBig5Done);
```

### UI表示

| 状態 | 表示内容 |
|------|---------|
| 両方未解析 | 「診断には自分と〇〇さん双方の性格解析（30回以上チャット）が必要です」 |
| 自分のみ未解析 | 「診断にはあなたの性格解析が必要です。チャットを30回以上続けてください」 |
| フレンドのみ未解析 | 「診断には〇〇さんの性格解析が必要です」 |
| 両方完了 | 警告非表示。「診断する」ボタン活性 |
| ロード中 | 警告非表示。「診断する」ボタン活性（誤表示防止） |

### Cloud Function側の処理

`diagnoseCompatibility` は `confirmedBig5Scores` → `convertedBig5Scores` の順でフォールバックしてスコアを取得する。どちらも存在しない場合は `{error: "BIG5データが不足しています"}` を返す。

### 実装ファイル

- `lib/presentation/screens/friend/compatibility_screen.dart`
- `lib/presentation/providers/character_provider.dart`（`userCharacterDetailsProvider`）

## フレンド一覧「?」ボタン（元素相性表）

### 概要

### 表示内容

- 元素相性表画像（`assets/images/element_chart.png`）全幅表示
- 説明テキスト: 「双方向矢印の元素同士は特に相性が良い対ペアです。無はすべての元素と一定の相性があります。」
- 対ペア4組の説明（`炎↔水` / `雷↔氷` / `光↔闇` / `風↔土`）

### 実装ファイル

- `lib/presentation/screens/friend/friend_screen.dart`（`_ElementChartSheet` ウィジェット）
