# UI/UX設計

## 🗺️ 画面遷移フロー

```
HomeView
├─ チャットで悩みを話す
│  └─ キャラクターが「6人に相談する？」と提案
│     └─ [6人の自分に相談する]ボタン
│        └─ MeetingScreen
│
└─ 右上の💭アイコンタップ
   └─ MeetingScreen（入力フェーズ）

MeetingScreen（フェーズ制）
├─ Phase: input         → 悩みテキスト入力
├─ Phase: introduction  → 6キャラクター紹介
├─ Phase: searching     → 検索中アニメーション
├─ Phase: conversation  → 会話表示（チャット形式）
└─ Phase: conclusion    → 統合レポート・評価
   ├─ [キャラクターと話す] → HomeView
   ├─ [TODOに追加] → TodoView
   ├─ [予定に追加] → CalendarView
   └─ [メモに保存] → MemoView
```

✅ **Flutter実装**: `flutter/lib/presentation/screens/meeting/meeting_screen.dart`

---

## 📱 画面詳細設計

### 1. ホーム画面の統合

- 右上に💭アイコンを追加し、タップでMeetingScreenを表示
- チャット内で悩みキーワードを検出した場合、「6人に相談する？」ボタンを表示
  - 検出キーワード例: 転職、仕事、辞めたい、恋愛、迷って、悩んで、不安、疲れた

---

### 2. 入力画面（input フェーズ）

- 悩みテキストエリア（自由入力）
- カテゴリは自動検出（サーバー側 `detectConcernCategory()`）
- 相談ボタン（テキスト空の場合はdisabled）

---

### 3. 6人会議画面（MeetingScreen）

#### 画面構成

```
┌─────────────────────────────────┐
│ ← 💭 6人会議                     │
├─────────────────────────────────┤
│                                 │
│  [Phase: introduction]          │
│  6キャラクターのプロフィール表示  │
│  → 「会議を始める」ボタン         │
│                                 │
│  [Phase: searching]             │
│  各キャラクターの調査中アニメーション│
│  進捗チェックマーク表示           │
│                                 │
│  [Phase: conversation]          │
│  チャット形式でメッセージ順次表示  │
│  左側: 慎重派 (青背景)            │
│  右側: 行動派 (オレンジ背景)      │
│  [結論へスキップ] ボタン          │
│                                 │
│  [Phase: conclusion]            │
│  統合レポート表示                 │
│                                 │
└─────────────────────────────────┘
```

---

### 4. 統合レポート（conclusion フェーズ）

✅ **実装済み** (`meeting_screen.dart` の `_buildConclusion()`)

実装済み内容：
- 結論サマリー（黄色背景）
- アドバイス（複数件・オレンジ背景）
- 次のステップ（複数件・青色背景）
- 共有ボタン（緑）
- ★5段階評価ダイアログ（`_showRatingDialog()`）

---

## 🎨 デザインガイドライン

### カラー
- 左側メッセージ（慎重派）: 青系
- 右側メッセージ（行動派）: オレンジ系

### アニメーション
- メッセージ表示間隔: 2.5〜5秒（メッセージ長に応じて動的調整）
  - ベース: 2.5秒
  - 追加: 1文字あたり0.017秒
  - 上限: 5秒
- メッセージのスライドイン（左右の方向に応じてフェードイン）
- スキップ機能で結論へジャンプ
