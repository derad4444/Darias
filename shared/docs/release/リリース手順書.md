# リリース手順書

> DARIASアプリのバージョン更新・リリース手順。Web版・iOS版それぞれの作業をまとめた手順書。

**最終更新日**: 2026-04-04

---

## バージョン番号の仕様

```
version: X.Y.Z+N
         ↑ ↑ ↑  ↑
         │ │ │  └── ビルド番号（+N）: リリースのたびに +1
         │ │ └───── パッチ（バグ修正のみ）
         │ └─────── マイナー（新機能追加）
         └───────── メジャー（大規模変更）
```

| バージョン例 | 使いどころ |
|------------|----------|
| `2.2.1+7` | バグ修正のみ |
| `2.3.0+7` | 新機能追加 |
| `3.0.0+7` | 大規模リニューアル |

---

## 変更が必要なファイル一覧

| ファイル | 変更箇所 | 備考 |
|--------|---------|------|
| `flutter/pubspec.yaml` | `version:` の値 | **必須・最初に変更** |
| `flutter/ios/Runner.xcodeproj/project.pbxproj` | `MARKETING_VERSION` (3箇所) と `CURRENT_PROJECT_VERSION` (3箇所) | Profile / Debug / Release の Runner 分のみ（RunnerTests の 1.0 / 1 は変更しない） |

> `flutter/ios/Flutter/Generated.xcconfig` は `flutter build ios --release` を実行すると **自動更新** されるため手動で変更しない。

## Web版リリース手順

### ① バージョン更新

```yaml
# flutter/pubspec.yaml
version: 2.3.0+7  # ← ここだけ変更
```

### ② ビルド

```bash
cd flutter
flutter build web --release
```

### ③ Firebase Hosting にデプロイ

```bash
# プロジェクトルート（DARIAS/）で実行
firebase deploy --only hosting:darias-app --project my-character-app
```

**公開URL**: https://darias-app.web.app

### ④ コミット＆プッシュ

```bash
git add flutter/pubspec.yaml
git commit -m "chore: バージョンをX.Y.Z+Nに更新"
git push origin main
```

## iOS版リリース手順

### ① バージョン更新（2箇所）

**1. `flutter/pubspec.yaml`**
```yaml
version: 2.3.0+7  # バージョン名+ビルド番号
```

**2. `flutter/ios/Runner.xcodeproj/project.pbxproj`**

以下の3箇所を変更（Runner の Profile / Debug / Release）:
```
MARKETING_VERSION = 2.3.0;      # X.Y.Z 部分（+N は含めない）
CURRENT_PROJECT_VERSION = 7;    # +N の数字
```

> ⚠️ `RunnerTests` の `MARKETING_VERSION = 1.0;` / `CURRENT_PROJECT_VERSION = 1;` は**変更しない**

### ② iOS ビルド（Generated.xcconfig を更新）

```bash
cd flutter
flutter build ios --release
```

> ⚠️ このステップを省略すると Xcode Archive 時に古いバージョンが使われる

### ③ Xcode で Archive

1. Xcode でプロジェクトを開く
2. `Product` → `Archive`
3. Archives に `X.Y.Z (N)` のアーカイブが作成されることを確認

### ④ App Store Connect にアップロード

1. Archives 画面で新しいアーカイブを選択
2. `Distribute App` → App Store Connect に提出

### ⑤ コミット＆プッシュ

```bash
git add flutter/pubspec.yaml flutter/ios/Runner.xcodeproj/project.pbxproj
git commit -m "chore: バージョンをX.Y.Z+Nに更新"
git push origin main
```

## Web + iOS 同時リリース手順（まとめ）

```
1. pubspec.yaml の version を更新
2. project.pbxproj の MARKETING_VERSION / CURRENT_PROJECT_VERSION を更新（3箇所）
3. flutter build web --release
4. firebase deploy --only hosting:darias-app --project my-character-app
5. flutter build ios --release
6. Xcode → Product → Archive → Distribute App
7. git add / commit / push
```

## よくあるミス

| 症状 | 原因 | 対処 |
|-----|------|------|
| Xcode Archive が古いバージョンになる | `flutter build ios --release` を実行していない → `Generated.xcconfig` が古いまま | `flutter build ios --release` を実行してから Archive |
| Xcode が「file modified by another application」ダイアログを出す | `project.pbxproj` を外部ツールで編集した | **「Use Version on Disk」** を選択 |
| Web版のバージョンが反映されない | `flutter build web --release` を再実行せずにデプロイした | ビルドしてからデプロイ |
